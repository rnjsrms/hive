# Hive Plugin for Claude Code

Multi-agent orchestration system that coordinates teams of agents to plan, implement, review, test, and validate code changes with enterprise quality.

Inspired by [Gastown](https://github.com/steveyegge/gastown), built with pure Claude Code Agent Teams.

## Quick Start

```bash
# Test locally
claude --plugin-dir ~/hive/plugins/hive

# Or install permanently
claude plugin install ~/hive/plugins/hive --scope user
```

When enabled, every `claude` session starts as the Hive lead orchestrator. The `"agent": "hive"` setting in `settings.json` replaces the default Claude Code agent with the Hive lead.

## How It Works

1. **Interview** — Hive asks clarifying questions about your objective
2. **Plan** — Creates a detailed plan with work items, risks, and acceptance criteria
3. **Team** — Spawns agent teammates: developers, reviewer, tester, researcher, monitor
4. **Execute** — Workers implement in isolated git worktrees on feature branches
5. **Validate** — Mandatory pipeline: code review → tests → ready-to-merge
6. **Complete** — Creates a PR from the feature branch to your base branch

## Architecture

| Agent | Role | Model | Worktree |
|-------|------|-------|----------|
| **hive** (lead) | Plans, coordinates, monitors — never writes code | opus | no |
| **hive-developer** | Implements features with enterprise quality | opus | yes |
| **hive-reviewer** | Reviews code, runs /simplify, challenges design | opus | yes |
| **hive-tester** | Writes tests, runs suites, validates correctness | opus | yes |
| **hive-researcher** | Explores codebase, researches APIs, finds patterns | opus | yes |
| **hive-monitor** | Health checks every 5 min, detects stuck agents | haiku | no |

## Branch Model

```
develop (base branch)
  └── feature/ABC-1234 (feature branch)
        ├── feature/ABC-1234_WI-1 (work item branch)
        ├── feature/ABC-1234_WI-2
        └── feature/ABC-1234_WI-3
```

- Base branch detection: `develop` → auto-detect → `main` (fallback)
- WI IDs are feature-scoped: `ABC-1234_WI-1` is unique globally
- Merge flow: WI → feature (lead, --no-ff) → base branch (user PR)
- All agents fetch latest before starting work

## Key Features

- **No subagents** — Hive lead IS the main session (via plugin agent replacement)
- **Agent teams** — Workers are full teammates with their own 1M context windows
- **Gitflow** — Agents only work on feature branches. Never touch main/develop directly.
- **Mandatory review+test** — Every work item MUST pass review AND testing before merge
- **Monitor always spawned** — Health checks run every 5 minutes automatically
- **Feature-scoped WI IDs** — WI numbers restart per feature for clean organization
- **All opus models** — Every agent except monitor uses the best available model
- **Persistent ledgers** — Communications and task state auto-logged by hooks
- **Hook-driven timestamps** — All timestamps generated by scripts, never by agents
- **State machine enforcement** — `validate-transition.sh` hook ensures valid status changes
- **Crash recovery** — Resume in-progress features on restart
- **Blocker escalation** — Timed ladder (15m re-ping → 30m user escalation)

## State

```
.hive/
├── config.json         # Project config (name, version, base_branch)
├── role-catalog.json   # Reviewer specializations
├── plans/              # Planning documents
├── research/           # Research findings
├── work-items/         # Work item state ({ticket-id}_WI-{id} format)
├── features/           # Feature groupings (with per-feature WI counter)
├── agents/             # Agent registry
├── archive/            # Completed/cancelled features
└── logs/               # Activity, communications, task ledger (JSONL)
```

## Enable/Disable

```bash
# Per-project
claude plugin enable hive --scope local
claude plugin disable hive --scope local

# Globally
claude plugin enable hive --scope user
claude plugin disable hive --scope user
```

## Requirements

- Claude Code with Agent Teams enabled (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`)
- Node.js 16+ (for hook scripts)
- Git
- Bash (Git Bash on Windows)
